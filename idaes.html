How AI, conceptual models, and UX can help novice users develop code

Background & Motivation - the PrOMMiS project

Stories about rare earth materials have been all over the news in recent years. Although these elements are not really that rare, refining them into usable products requires extracting and processing them from raw materials, and these capabilities are much more scarce. Building the infrastructure for such processes demands significant time and investment. Traditionally, engineers have relied on prototypes and physical models built at multiple scales to test and validate new processing systems before full-scale implementation.

Computational simulation and optimization can help. The PrOMMiS project is a multi-institutional collaboration building a software modeling platform to accelerate development and deployment of extraction and purification processes for critical minerals and rare earth elements. The core software is a python based library that can be used to represent these systems as mathematical equations. This allows computational methods such as optimization to help engineers and planning understand the system.

Although these methods are powerful, it comes with a steep learning curve. Many seasoned process engineers may not be well versed in programming and also computational optimization methods. Over numerous interviews, workshops, and meetings, challenges with learning such libraries and understanding and debugging the models have been consistent themes. Creating a drag and drop UI could help in terms of building models, but the models themselves can feel like a bit of a black box that is difficult to fully inspect and understand.

AI and Conceptual models as powerful partners for non coders

Using agenic AI to help write code is increasingly becoming a common practice, especially when working with unfamiliar codes and libraries. Responding to a prompt, AI agents will commonly provide a narrative description of what changes they suggest and why they chose those changes. When integrated with a IDE such as VS Code, they may also provide code diffs so you can visually verify the changes.

Using agenic AI will potentially be a great leap in terms of helping new users work with the PrOMMiS framework. However, showing the code diffs might not be enough to help users understand and verify the changes the agent is making on their behalf.

This had me reflecting on my experiences using agenic AI to update frontend web code . Even if I didn't really understand the code changes that were being suggested, I could view the UI updates to know that the changes made were what I intended. Having a visual representation of the interpreted code - or a visual conceptual model that serves as an abstract representation of a system - is an important bridge in helping people use AI to develop code.

Early UX ideation on a conceptual model design for PrOMMiS

There's still a lot of work to be done, but here's a sneak peak at what a conceptual model for PrOMMiS could look like. When coming up with a visual conceptual model, I focused on the use cases that were common among novice users and areas where they typically struggled the most, based on interviews with new users as with mentors who frequently work with new users. Namely, I knew that new users commonly struggled with changing the variables and constraints of the system without violating the bounds of the system. With this in mind, I had a few design goals. I wanted:

  • A domain-specific UI that could summarize for users the main concepts and relationships in the system.
  • To show how the visual could be integrated into a developer tool that uses an AI agent.
  • A way to highlight which aspects have been changed by the AI agent for inspection and verification.
